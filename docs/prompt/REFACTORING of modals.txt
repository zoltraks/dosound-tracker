Looking at `App.tsx`, it's definitely grown quite large (over 1500 lines), and moving the modal dialogs to separate components would significantly improve maintainability. 

Here's proposed refactoring:

## Recommended Refactoring Strategy

### 1. **Create a `modals` directory structure**

```
src/
├── modals/
│   ├── AboutModal.tsx
│   ├── ChangelogModal.tsx
│   ├── ConfirmDialog.tsx (reusable)
│   ├── InfoDialog.tsx (reusable)
│   ├── NewSongConfirmModal.tsx
│   ├── ResetConfirmModal.tsx
│   ├── OptimizeModal.tsx
│   ├── RenumberModal.tsx
│   ├── TransposeModal.tsx
│   ├── ExportSummaryModal.tsx
│   ├── ErrorModal.tsx
│   └── index.ts (barrel export)
```

### 2. **Create reusable base components**

Since many of your modals share similar structure, create these base components:

**`modals/InfoDialog.tsx`** - For simple info/summary modals:
```tsx
interface InfoDialogProps {
  isOpen: boolean;
  title: string;
  message: string;
  onClose: () => void;
}

export const InfoDialog: React.FC<InfoDialogProps> = ({
  isOpen,
  title,
  message,
  onClose
}) => {
  if (!isOpen) return null;

  return (
    <div className="modal-backdrop">
      <div className="modal-dialog">
        <div className="modal-title">{title}</div>
        <div className="modal-body">
          {message.split('\n').map((line, index) => (
            <React.Fragment key={index}>
              {line}
              {index < message.split('\n').length - 1 && <br />}
            </React.Fragment>
          ))}
        </div>
        <div className="modal-actions">
          <button className="command-btn" onClick={onClose}>
            OK
          </button>
        </div>
      </div>
    </div>
  );
};
```

**`modals/ConfirmDialog.tsx`** - For confirmation dialogs:
```tsx

```

### 4. **Consolidate modal state management**

Create a custom hook to manage all modal states:

**`hooks/useModals.ts`**:
```tsx
export const useModals = () => {
  const [aboutOpen, setAboutOpen] = useState(false);
  const [changelogOpen, setChangelogOpen] = useState(false);
  const [newSongConfirmOpen, setNewSongConfirmOpen] = useState(false);
  // ... all other modal states

  return {
    about: {
      isOpen: aboutOpen,
      open: () => setAboutOpen(true),
      close: () => setAboutOpen(false)
    },
    changelog: {
      isOpen: changelogOpen,
      open: () => setChangelogOpen(true),
      close: () => setChangelogOpen(false)
    },
    // ... etc
  };
};
```

## Benefits of This Refactoring

1. **Reduced file size**: App.tsx drops from ~1500 lines to ~800-900 lines
2. **Better testability**: Each modal can be tested independently
3. **Reusability**: `InfoDialog` and `ConfirmDialog` eliminate duplicate code
4. **Maintainability**: Changes to modal behavior are isolated
5. **Easier navigation**: Finding modal-related code is straightforward
6. **Type safety**: Each modal has explicit TypeScript interfaces

## Migration Strategy

1. Start with the simplest modals (error/info dialogs)
2. Move to confirmation dialogs
3. Tackle complex modals like TransposeModal last
4. Extract the keyboard event handler logic into a separate hook
