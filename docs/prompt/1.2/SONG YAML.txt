There is small inconsequence when saving song in YAML format.
In the "pattern" definition block value for "number" key is written
in double quotes when its value < 16 and hexadecimal representation has only digits,
and without double quotes when it contains letters.
Always write value for "number" with double quotes.

```yaml
  pattern:
    - number: "01"
      step:
        - wait: 2
        - note: "E-2"
          instrument: "01"
        - wait: 1
    - number: 0B
      step:
        - note: "C-3"
          instrument: "00"
        - wait: 5
        - note: "C-3"
          instrument: "00"
        - wait: 5
        - note: "C-4"
          instrument: "03"
```

When saving song in YAML write value for "title" always in double quotes like this:

```yaml
song:
  title: "TEST A-4"
```

---

When saving song with title "TEST A-4" the download file name is "TEST_A_4.yaml".
Make it to be "TEST_A_4.song.yaml" (.song.yaml instead of .yaml as extension).
But when song has no title, then file is just "yaml" and that's wrong.
Assume that if there is no title, the download file should be just "NONAME.song.yaml".

When saving instrument in YAML the download name is formatted in lower case.
Make it similar how the name of the song is made.
Keep "Instrument_XX" as base file name if instrument has no name set.
Use ".inst.yaml" as file extension instead of just ".yaml".

When saving song in YAML format add two additional fields "chip" and "format" with default values.

This is part of current song YAML:

```yaml
song:
  title: "Warmball"
  author: Zoltar X / New Generation
  year: 2025
  speed: 4
  length: 32
  loop: 0
```

Extended version:

```
song:
  title: "Warmball"
  author: Zoltar X / New Generation
  year: 2025
  chip: YM
  frame: 50
  speed: 4
  length: 32
  loop: 0
```

When loading song YAML and "chip" or "frame" keys are present
perform simple check if they have supported values: "YM" (case insensitive) for "chip", 50 for "frame".
If not, show confirmation modal dialog before loading with the warning
and allow user to proceed ("OK") or not ("CANCEL").
